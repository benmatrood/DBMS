DBMS ET LE LANGUAGE SQL

Qu'est ce que le DBMS

Le système de gestion de base de données ou DBMS fait référence à la technologie de stockage et de récupération des données des utilisateurs avec la plus grande efficacité ainsi que des mesures de sécurité appropriées. Ce didacticiel explique les bases du DBMS telles que son architecture, les modèles de données, les schémas de données, l'indépendance des données,le modèle de relation, la conception de base de données relationnelle, le stockage et la structure des fichiers et bien plus encore

Applications du DBMS

La base de données est une collection de données connexes et les données sont une collection de faits et de chiffres qui peuvent être traités pour produire des informations.La plupart des données représentent des faits enregistrables. Les données aident à produire des informations basées sur des faits. Par exemple, si nous avons des données sur les notes obtenues par tous les élèves, nous pouvons alors conclure sur les meilleurs et les notes moyennes.Un système de gestion de base de données stocke les données de telle manière qu'il devient plus facile de récupérer, manipuler et produire des informations. Voici les caractéristiques et applications importantes du DBMS.

1) Spécificité

Un DBMS moderne est plus réaliste et utilise des entités du monde réel pour concevoir son architecture. Il utilise également le comportement et les attributs. Par exemple, une base de données scolaire peut utiliser les élèves comme une entité et leur âge comme un attribut.

2) Accès multi-utilisateur et simultané

Le DBMS prend en charge l'environnement multi-utilisateur et leur permet d'accéder et de manipuler des données en parallèle. Bien qu'il existe des restrictions sur les transactions lorsque les utilisateurs tentent de traiter le même élément de données, mais les utilisateurs n'en sont toujours pas conscients.

3) Vues multiples

Le DBMS offre plusieurs vues pour différents utilisateurs. Un utilisateur qui est dans le département des ventes aura une vue de la base de données différente de celle d'une personne travaillant dans le département de production. Cette fonctionnalité permet aux utilisateurs d'avoir une vue concentrée de la base de données en fonction de leurs besoins.

4) La Sécurité

Des fonctionnalités telles que les vues multiples offrent une sécurité dans une certaine mesure où les utilisateurs ne peuvent pas accéder aux données d'autres utilisateurs et services. Le DBMS propose des méthodes pour imposer des contraintes lors de la saisie des données dans la base de données et de leur récupération à un stade ultérieur.

Les types de DBMS

Il existe plus de 300 systèmes répartis en 5 niveaux .

Niveau 1: omniprésent, utilisé partout et par tous : DBMS relationnel (RDBMS)

Exemples de systèmes: Microsoft Access et MySQL Exemples d'utilisateurs: Codebots (et 99% des applications et sites Web) Un RDBMS est comme un tableur. Les données sont organisées en classeurs de tables contenant des colonnes et des lignes. La terminologie d'un RDBMS est des bases de données d'entités caractérisées par leurs attributs et leurs enregistrements. Les schémas sont la caractéristique déterminante des bases de données relationnelles. Les schémas définissent les relations entre les entités et la mise en forme dans les enregistrements d'attributs.

Niveau 2: Magasins de documents

Exemples de systèmes: MongoDB Exemples d'utilisateurs: Scratchpad d'Expedia Aussi connus sous le nom de systèmes de base de données orientés document, les magasins de documents (comme les feuilles de calcul) n'ont pas de schéma. Cela signifie que les enregistrements peuvent être non uniformes et / ou avoir une structure imbriquée et que les valeurs peuvent être stockées sous forme de tableaux. Les magasins de documents sont un exemple majeur de base de données NoSQL. Ce type de DBMS gagne en popularité à mesure que des solutions à des défis spécifiques de gestion des données se présentent.

Niveau 3: DBMS graphique

Exemples de systèmes: Neo4j et Microsoft Azure Cosmos DB Exemple d'utilisateurs: Ebay Aussi connu sous le nom de DBMS orienté graphe ou base de données de graphes, un DBMS graphe est basé autour de structures de nœuds et d'arêtes. Les arêtes définissent les relations entre les nœuds. Ces bases de données sont conçues pour rendre efficacement des représentations visuelles des données.

Niveau 4: DBMS orienté objet

Exemples de systèmes: Db4o Un DBMS orienté objet suit un modèle de données orienté objet avec des classes, des propriétés et des méthodes. Un objet est toujours géré dans son intégralité. Cela signifie, par exemple, que la création d'un objet, qui dans un système relationnel serait probablement stocké sur plusieurs tables, sera effectuée automatiquement, comme une transaction atomique, et sans aucune action du programme d'application.

Niveau 5: DBMS de navigation

Exemples de systèmes: IMS et IDMS Le terme DBMS de navigation décrit une classe de systèmes de gestion de base de données dans laquelle les données ne sont accessibles que via des enregistrements liés. Ce modèle a été développé dans les années 1960 et a été le premier DBMS capable de gérer de grandes quantités de données. En fonction de la flexibilité de la liaison, les systèmes peuvent être sous-regroupés en DBMS hiérarchiques et DBMS réseau. Ces deux sous-classes ont été largement remplacées par le RDBMS.
